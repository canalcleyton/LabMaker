1-Prompt de comando como Administrador

Existem várias maneiras de abrir o Prompt de Comando como administrador no Windows. Aqui estão as formas mais comuns:

Método 1: Menu Iniciar (Pesquisa)

  1. Clique no botão Iniciar (ou pressione a tecla Windows).
  2. Digite cmd ou prompt de comando na caixa de pesquisa.
  3. Nos resultados da pesquisa, você verá "Prompt de Comando". Clique com o botão direito do mouse sobre ele.
  4. No menu de contexto que aparece, selecione "Executar como administrador".
  5. Se o Controle de Conta de Usuário (UAC) perguntar se você deseja permitir que o aplicativo faça alterações no dispositivo, clique em "Sim".

Método 2: Menu Iniciar (Lista de Aplicativos)

  1. Clique no botão Iniciar.
  2. Role para baixo na lista de aplicativos até encontrar a pasta "Sistema do Windows" (ou "Windows System" em inglês).
  3. Expanda a pasta "Sistema do Windows".
  4. Clique com o botão direito do mouse em "Prompt de Comando".
  5. Selecione "Mais" e, em seguida, "Executar como administrador".
  6. Clique em "Sim" no prompt do UAC, se solicitado.

Método 3: Gerenciador de Tarefas

  1. Pressione Ctrl + Shift + Esc (ou Ctrl + Alt + Del e selecione "Gerenciador de Tarefas") para abrir o Gerenciador de Tarefas.
  2. Clique em "Arquivo" -> "Executar nova tarefa".
  3. Na caixa "Abrir", digite cmd.
  4. Marque a caixa de seleção que diz "Criar esta tarefa com privilégios administrativos".
  5. Clique em "OK".
  6. Clique em "Sim" no prompt do UAC.

Método 4: Explorador de Arquivos

  1. Abra o Explorador de Arquivos (Windows + E).
  2. Navegue até a pasta C:\Windows\System32.
  3. Encontre o arquivo cmd.exe.
  4. Clique com o botão direito do mouse em cmd.exe.
  5. Selecione "Executar como administrador".
  6. Clique em "Sim" no UAC.

Método 5: Atalho na Área de Trabalho (Opcional - para uso frequente)

Se você usa o Prompt de Comando como administrador com frequência, pode criar um atalho na área de trabalho que sempre o abra com privilégios elevados:

  1. Clique com o botão direito em uma área vazia da área de trabalho.
  2. Selecione "Novo" -> "Atalho".
  3. No campo "Digite o local do item", digite cmd.exe.
  4. Clique em "Avançar".
  5. Dê um nome ao atalho (por exemplo, "Prompt de Comando (Admin)").
  6. Clique em "Concluir".
  7. Agora, clique com o botão direito do mouse no novo atalho que você criou.
  8. Selecione "Propriedades".
  9. Vá para a guia "Atalho".
  10. Clique no botão "Avançado...".
  11. Marque a caixa de seleção que diz "Executar como administrador".
  12. Clique em "OK" em todas as janelas abertas.
  13. Agora, sempre que você clicar duas vezes nesse atalho, o Prompt de Comando será aberto como administrador.

Qual método escolher?

  • Menu Iniciar (Pesquisa): É o método mais rápido e fácil para a maioria das situações.
  • Gerenciador de Tarefas: Útil se você já tiver o Gerenciador de Tarefas aberto ou se o menu Iniciar estiver com problemas.
  • Atalho na Área de Trabalho: Ideal para quem usa o Prompt de Comando como administrador regularmente.

Independentemente do método escolhido, o resultado será o mesmo: um Prompt de Comando com privilégios elevados, permitindo que você execute comandos que exigem permissões de administrador. Você saberá que está executando como administrador porque o título da janela do Prompt de Comando começará com "Administrador:".

2-Instalação do 7-Zip no Windows

  1. Download:

    • Vá para o site oficial do 7-Zip: https://www.7-zip.org/
    • Na seção "Download", você verá várias opções. Escolha a versão correta para o seu sistema:
      • 64-bit x64: Para a maioria dos computadores modernos com Windows de 64 bits.
    • Clique no link de download correspondente (geralmente um arquivo .exe).

  2. Execução do Instalador:

    • Localize o arquivo baixado (normalmente na pasta "Downloads").
    • Clique duas vezes no arquivo .exe para iniciar o instalador.

  3. Controle de Conta de Usuário (UAC):

    • Se o Windows solicitar permissão para executar o instalador, clique em "Sim".

  4. Assistente de Instalação:

    • Pasta de Destino:

      • O instalador sugerirá uma pasta padrão (geralmente C:\Program Files\7-Zip). Você pode alterá-la se desejar, mas a padrão é recomendada para a maioria dos usuários.
      • Clique em "Install".

    • Progresso:

      • Aguarde enquanto o 7-Zip é instalado. É um processo rápido.

    • Conclusão:

      • Quando a instalação terminar, clique em "Close".

  5. Verificação (Opcional):

    • Abra o Explorador de Arquivos (Windows + E).
    • Clique com o botão direito em qualquer arquivo ou pasta.
    • Você deverá ver as opções do 7-Zip no menu de contexto (ex: "7-Zip" -> "Adicionar para arquivo...", "Extrair aqui", etc.). Isso indica que a instalação foi bem-sucedida.

  6. Configurações (Opcional, mas Recomendado):

    • Abra o 7-Zip File Manager (você pode procurá-lo no menu Iniciar).
    • Vá para "Ferramentas" -> "Opções".
    • Aba "Sistema":
      • Aqui você pode associar tipos de arquivos ao 7-Zip. Marque as caixas dos formatos que você deseja que o 7-Zip abra por padrão (ex: .zip, .rar, .7z, etc.).
    • Aba "7-Zip":
      • Configure outras opções, se quiser, como o comportamento do menu de contexto.
    • Clique em "OK" para salvar as alterações.

3-Tutorial de Instalação do Driver Serial Virtual com0com (versão assinada) em C:\Store\Programas

Este tutorial usa a versão com0com-3.0.0.0-i386-and-x64-signed.zip e assume que você está executando comandos em um prompt de comando com privilégios de administrador.

  1. Fazer o download do arquivo em zip: com0com-3.0.0.0-i386-and-x64-signed.zip

Endereço para baixar o arquivo:

https://sourceforge.net/projects/com0com/files/com0com/3.0.0.0/

ou pelo link direto:

https://sourceforge.net/projects/com0com/files/com0com/3.0.0.0/com0com-3.0.0.0-i386-and-x64-signed.zip/download

  1. Criação do Diretório:

    • Abra um prompt de comando como administrador.

    • Crie o diretório de destino:

      mkdir C:\Store\Programas

  2. Extração:

    • Vá para a pasta onde você baixou o arquivo .zip.
    • Extraia o conteúdo do .zip, diretamente para C:\Store\Programas. Você pode usar:

      • Explorador de Arquivos: Clique com o botão direito no arquivo .zip, "Extrair Tudo...", e defina o destino como C:\Store\Programas.

      • Linha de Comando (exemplo com 7-Zip): Se você tiver o 7-Zip instalado e configurado no seu PATH, você pode usar:

        cd /d "C:\Caminho\Para\Onde\Voce\Baixou" 
7z x com0com-3.0.0.0-i386-and-x64-signed.zip -o C:\Store\Programas

        (Ajuste os caminhos conforme necessário, mas recomenda-se a pasta C:\Store\Programas).

      • Estrutura de Diretórios (IMPORTANTE): Após a extração, você deve ter a seguinte estrutura (ou algo muito similar):

        C:\Store\Programas\
    com0com-3.0.0.0\  (o nome exato da pasta pode variar um pouco)
        setupc.exe
        setupg.exe
        ... (outros arquivos e pastas)

        O setupc.exe e setupg.exe devem estar dentro de uma subpasta, não diretamente em C:\Store\Programas.

  3. Instalação via Linha de Comando (setupc.exe):

    • No prompt de comando como administrador, navegue até a pasta onde você extraiu o com0com:

      cd /d C:\Store\Programas\com0com-3.0.0.0

      (Ajuste o nome da pasta com0com-3.0.0.0 se necessário).

    • Execute os comandos de instalação. Os exemplos do Tutorial anterior ainda se aplicam:

      • Instalação padrão (par de portas):

        setupc.exe install

      • Nomes de portas personalizados:

        setupc.exe install PortName=COM3 PortName=COM4

        (Substitua COM3 e COM4 pelos nomes desejados).

      • Listar portas:

        setupc.exe list

      • Remover uma porta:

        setupc.exe remove PortName=COM4

      • Remover todas as portas:

        setupc.exe uninstall

  4. Verificação:

    • Abra o Gerenciador de Dispositivos (procure no menu Iniciar).
    • Expanda "Portas (COM e LPT)".
    • Verifique se as portas virtuais que você criou estão listadas.

4-Tutorial de Instalação do SimulIDE 1.1.0 (Portable)

1. Baixar o SimulIDE 1.1.0

  1. Acesse o link de download:
    📎 https://simulide.com/p/download110/
  2. Escolha a versão correspondente ao seu sistema operacional (Windows). (Geralmente Windows 64)
  3. Responder 0 para: I will contribute with a donation of *
  4. Faça o download do arquivo compactado (SimulIDE_1.1.0_win64.zip ou similar).

2. Mover o arquivo para o local desejado

  1. Após o download, localize o arquivo baixado (geralmente na pasta Downloads).
  2. Mova o arquivo .zip para a pasta de destino: 
    C:\Store\Programas

3. Extrair os arquivos usando o 7-Zip

  1. Abra o Explorador de Arquivos (Win + E).
  2. Navegue até C:\Store\Programas.
  3. Clique com o botão direito no arquivo SimulIDE_1.1.0_win64.zip.
  4. No menu de contexto, selecione 7-Zip → Extrair aqui.
    • Isso criará uma pasta chamada SimulIDE_1.1.0_win64 (ou similar).

💡 Alternativamente, você pode extrair para uma pasta específica:

  • Escolha 7-Zip → Extrair para "SimulIDE_1.1.0_win64"
  • Isso criará uma subpasta dentro de C:\Store\Programas para manter os arquivos organizados.

4. Acessar e Executar o SimulIDE

  1. Após a extração, abra a pasta: 
    C:\Store\Programas\SimulIDE_1.1.0_win64
  2. Localize o executável SimulIDE.exe.
  3. Clique duas vezes para abrir o SimulIDE.

5. Criar um Atalho na Área de Trabalho (Opcional)

Para facilitar o acesso ao SimulIDE:

  1. Clique com o botão direito em SimulIDE.exe.
  2. Selecione Enviar para → Área de Trabalho (Criar atalho).
  3. Agora você pode abrir o SimulIDE diretamente da área de trabalho.

5-Tutorial: Configurando o Ambiente Arduino (com arquivos ZIP)

Este tutorial irá guiá-lo passo a passo para configurar o ambiente de desenvolvimento Arduino (IDE) utilizando arquivos ZIP baixados, garantindo que tudo funcione corretamente.

Pré-requisitos:

  • Você já baixou os dois arquivos ZIP fornecidos nos links.
  • Você tem permissões de administrador no seu computador (para instalar programas).

Passos:

1. Instalando o Ambiente Arduino (IDE):

  • Extraia o arquivo ZIP do Ambiente Arduino:
    • Localize o arquivo ZIP que você baixou do primeiro link (https://drive.google.com/file/d/1EdfAg_awdkqBdC9F_bpD5J5uI2LlkhxO/view?usp=sharing). O nome deve ser arduino-1.8.19.zip (o nome real pode ser diferente).
    • Mova o arquivo baixado para C:\Store\Programas\ (Ao selecionar o arquivo arduino-1.8.19.zip, pressione as duas teclas do teclado juntas: Ctrl + X e depois vá para basta C:\Store\Programas\ e pressione as duas teclas do teclado juntas Ctrl+V)
    • Depois Clique com o botão direito do mouse no arquivo arduino-1.8.19.zip e selecione "Extrair Tudo...", clicando nesta opção com o botão esquerdo.
    • Dentro de C:\Store\Programas\Arduino você deverá ver arquivos como arduino.exe, pastas como lib, hardware, etc.

2. Instalando as Bibliotecas:

  • Extraia o arquivo ZIP das bibliotecas:
    • Localize o arquivo ZIP que você baixou do segundo link (https://drive.google.com/file/d/1E7TTjMU5mzxaA-oCObdOpCSLnoCjJSAu/view?usp=sharing). O nome deve ser Arduino.zip (o nome real pode ser diferente).
    • Clique com o botão direito no arquivo Arduino.zip e selecione "Extrair Tudo...".
    • Na janela de extração, clique em "Procurar..." e navegue até a pasta C:\Users\<nomeusuario>\Documentos\.
    • Clique em "Selecionar Pasta" e depois em "Extrair". Isso irá extrair as bibliotecas para a pasta correta. Dentro de C:\Users\<nomeusuario>\Documentos\Arduino\libraries, você deverá ver várias pastas, cada uma representando uma biblioteca (por exemplo, LiquidCrystal, Servo, etc.). Importante: Cada biblioteca deve estar em sua própria pasta diretamente dentro da pasta libraries. Não deve haver subpastas desnecessárias. Por exemplo, o caminho correto para a biblioteca LiquidCrystal seria C:\Users\<nomeusuario>\Documentos\Arduino\libraries\LiquidCrystal, e não C:\Users\<nomeusuario>\Documentos\Arduino\libraries\algumapasta\LiquidCrystal.

3. Configurando o Arduino IDE (se necessário):

  • Inicie o Arduino IDE:
    • Vá para C:\Store\Programas\Arduino e dê um duplo clique em arduino.exe para iniciar o programa.
  • Verifique o local da pasta de Sketchbook (Opcional, mas recomendado):
    • No Arduino IDE, vá em Arquivo -> Preferências.
    • Verifique se o campo "Localização da pasta de Sketchbook" está definido como C:\Users\<nomeusuario>\Documentos\Arduino. Se não estiver, clique em "Procurar...", navegue até essa pasta e clique em "Selecionar Pasta".
    • Clique em "OK".
    • Feche e reabra o Arduino IDE para garantir que as alterações sejam aplicadas. Isso garante que o IDE procure as bibliotecas e os exemplos no local correto.

4. Testando a Instalação:

  • Abra um exemplo:
    • No Arduino IDE, vá em Arquivo -> Exemplos -> 01.Basics -> Blink.
  • Selecione a placa e a porta:
    • Vá em Ferramentas -> Placa e selecione o tipo de placa Arduino que você está usando (por exemplo, "Arduino Uno"). Esta informação deve estar no arquivo ZIP do ambiente ou na documentação da sua placa.
    • Vá em Ferramentas -> Porta e selecione a porta COM à qual seu Arduino está conectado. Se você não tiver certeza, desconecte e reconecte o Arduino e observe qual porta aparece e desaparece.
  • Carregue o código:
    • Clique no botão "Carregar" (a seta apontando para a direita).
  • Verifique o resultado:
    • Se a instalação estiver correta, o LED embutido na sua placa Arduino (geralmente no pino 13) começará a piscar. Se houver erros, revise os passos anteriores, especialmente a estrutura das pastas e os nomes das pastas das bibliotecas.

Solução de Problemas:

  • Erro "arquivo .h não encontrado": Isso geralmente significa que a biblioteca não está instalada corretamente ou que o IDE não está procurando no local correto. Verifique se a biblioteca está na pasta libraries e se o "Localização da pasta de Sketchbook" está definido corretamente. Certifique-se também de que não há subpastas extras dentro da pasta da biblioteca.
  • Erro de compilação: Verifique se você selecionou a placa e a porta corretas. Se o erro persistir, pode haver um problema com o código do exemplo ou com a instalação do ambiente Arduino.
  • A porta COM não aparece: Certifique-se de que o Arduino está conectado corretamente ao computador e que os drivers estão instalados. Reinicie o computador, se necessário. Em alguns casos, pode ser necessário instalar drivers manualmente (consulte a documentação do seu Arduino).
  • Erros relacionados a hardware específico dentro do arquivo zipado: Se o arquivo ZIP do ambiente Arduino (o primeiro link) contiver configurações para hardware específico (por exemplo, uma placa customizada), pode ser necessário copiar a pasta hardware desse ZIP para dentro da pasta C:\Users\<nomeusuario>\Documentos\Arduino\. Isso sobrescreverá ou complementará a pasta hardware que já existe lá. Faça isso apenas se o ZIP do ambiente Arduino explicitamente instruir.

Este tutorial detalhado deve ajudá-lo a configurar o ambiente Arduino com sucesso. Se você encontrar algum problema, revise cuidadosamente os passos e as dicas de solução de problemas. Boa sorte com seus projetos Arduino!
